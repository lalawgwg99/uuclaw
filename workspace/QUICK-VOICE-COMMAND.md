# 快速語音命令

## 臨時解決方案

由於自動檢測可能還需要調整，你可以使用以下命令手動觸發語音回覆：

### 命令 1: 要求語音回覆
在 Telegram 中發送：
```
/voice <你的問題>
```

例如：
```
/voice 今天天氣怎麼樣？
```

Agent 會用語音回覆。

### 命令 2: 測試語音
```
測試語音
```

Agent 會立即發送一個測試語音給你。

### 命令 3: 明確要求
```
用語音回覆我：<你的問題>
```

## 為什麼自動檢測可能不工作

OpenClaw 可能在接收語音消息時就已經轉換成文字了（STT - Speech to Text），所以 Agent 看到的都是文字，無法區分原始消息類型。

## 下一步

需要修改 OpenClaw 的核心代碼，在消息元數據中保留 "原始消息類型" 標記，這樣 Agent 才能知道用戶發送的是語音還是文字。

## 當前可用方案

1. **手動觸發**：明確告訴 Agent "用語音回覆"
2. **測試命令**：說 "測試語音" 來驗證功能
3. **等待修復**：需要修改 OpenClaw 核心代碼來支持自動檢測
